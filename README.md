# Fsociety_decoder Chat

Real-time chat website with:
- Direct Login/Register step
- Red text on black background style
- Socket.IO chat for people in different locations
- 24/7 runtime config using PM2

## Run locally

1. Install dependencies:
   ```bash
   npm install
   ```
2. Create `.env` from `.env.example` and set your values.
3. Start:
   ```bash
   npm start
   ```
4. Open: `http://localhost:3000`

## Phone access (same Wi-Fi)

1. Find your PC local IP (example `192.168.1.25`).
2. Start app with `HOST=0.0.0.0` (already default in `.env.example`).
3. Allow Node.js through Windows Firewall (Private network).
4. On phone open: `http://YOUR_PC_IP:3000`

## Deploy on Render (public HTTPS, no tunnel)

1. Push this project to a GitHub repo.
2. In Render, create a new **Blueprint** and connect that repo.
3. Render will detect `render.yaml` and create the web service.
4. After deploy, open your public URL: `https://<your-service>.onrender.com`

Required environment values in Render:
- `JWT_SECRET` (auto-generated by `render.yaml`, rotate if needed)
- `CORS_ORIGIN` (leave empty when frontend and backend are same Render URL)

Notes:
- This app serves frontend + API from one service, so login and chat use HTTPS by default.
- Render free services can sleep when idle.
- File storage on Render is ephemeral, so `data/users.json` and `data/messages.json` reset on redeploy/restart unless you move to a database.

## 24/7 run (recommended)

Install PM2 globally:
```bash
npm install -g pm2
```

Start app:
```bash
pm2 start ecosystem.config.js
```

Save PM2 process list:
```bash
pm2 save
```

Enable auto start on boot:
```bash
pm2 startup
```
Then run the command PM2 prints for your system.
